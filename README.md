# Customer Churn Prediction

A complete end-to-end machine learning pipeline that predicts customer churn for a telecom company. The project covers data cleaning, exploratory data analysis, feature engineering, model training (Logistic Regression, Decision Tree, Random Forest), and model interpretation using feature importance.

## Project Overview

Customer churn — the loss of clients to competitors — is a critical business problem in the telecom industry. This project builds a supervised binary classification pipeline to identify customers who are likely to churn, enabling proactive retention strategies.

**Target variable:** `churn` (0 = retained, 1 = churned)  
**Best model:** Random Forest Classifier  
**Dataset size:** 3,333 records, 20 raw features

---

## Dataset

The raw dataset (`churn_dataset.csv`) is the classic telecom customer churn dataset. It contains account information, usage statistics, and service plan details for telecom customers across US states.

| Column | Description |
|---|---|
| `state` | US state abbreviation |
| `account_length` | Number of days the account has been active |
| `area_code` | Customer area code |
| `international_plan` | Whether the customer has an international plan (Yes/No) |
| `voice_mail_plan` | Whether the customer has a voicemail plan (Yes/No) |
| `number_vmail_messages` | Number of voicemail messages |
| `total_day_minutes` | Total minutes used during the day |
| `total_day_calls` | Total number of day calls |
| `total_day_charge` | Total charge for day calls |
| `total_eve_minutes` | Total evening minutes |
| `total_eve_calls` | Total evening calls |
| `total_eve_charge` | Total evening charge |
| `total_night_minutes` | Total night minutes |
| `total_night_calls` | Total night calls |
| `total_night_charge` | Total night charge |
| `total_intl_minutes` | Total international minutes |
| `total_intl_calls` | Total international calls |
| `total_intl_charge` | Total international charge |
| `customer_service_calls` | Number of calls to customer service |
| `churn` | Target variable — True if churned, False if retained |

Place the raw CSV at `data/raw/churn_dataset.csv`.

## Notebooks

The five notebooks form a sequential pipeline. Run them in order.

### 01 — Data Cleaning (`01_data_cleaning.ipynb`)

Loads the raw dataset and performs the following preprocessing steps:

- **Initial inspection** — shape, dtypes, null values, duplicates
- **Column standardisation** — all column names converted to lowercase snake_case
- **Type conversion** — `International plan` and `Voice mail plan` binary-encoded (Yes→1, No→0); `Churn` cast to integer
- **Whitespace stripping** — trailing/leading spaces removed from all string columns
- **Output** — cleaned dataset saved to `data/processed/cleaned_churn_dataset.csv`

### 02 — Exploratory Data Analysis (`02_eda.ipynb`)

Analyses distributions and relationships to surface churn signals:

- Churn class distribution (2,850 retained vs 483 churned — ~14.5% churn rate)
- Box plots for account length, day/evening/night minutes by churn label
- Count plots for categorical features (international plan, voicemail plan, customer service calls)
- Pearson correlation heatmap across all numeric features
- Summary of key insights (see [Key Findings](#key-findings))

### 03 — Feature Engineering & Model Preparation (`03_feature_engineering.ipynb`)

Prepares features for modelling:

- **Drops redundant columns** — charge columns (`total_day_charge`, `total_eve_charge`, `total_night_charge`, `total_intl_charge`) are perfectly correlated with their corresponding minute columns and are removed
- **Drops non-predictive columns** — `state` and `area_code` carry no meaningful signal for individual churn behaviour
- **Train/test split** — 80/20 stratified split (`random_state=42`) yielding 2,666 training and 667 test samples across 13 features

### 04 — Model Training & Evaluation (`04_model_training.ipynb`)

Trains and compares three classifiers:

| Model | Notes |
|---|---|
| Logistic Regression | Linear baseline |
| Decision Tree | Captures non-linear splits; prone to overfitting |
| Random Forest | Ensemble of 100 trees; best overall performance |

Evaluation metrics reported per model: accuracy, precision, recall, F1-score, and confusion matrix.

**Random Forest was selected as the final model** based on the most balanced precision/recall trade-off.

### 05 — Model Interpretation & Feature Importance (`05_model_interpretation.ipynb`)

Extracts and visualises feature importances from the trained Random Forest to explain which variables drive churn predictions. Top features identified: `total_day_minutes` and `customer_service_calls`.

## Source Scripts

The `src/` directory contains production-ready Python scripts that replicate the notebook pipeline end-to-end without requiring Jupyter.

### `src/train_model.py`

Reads the processed dataset, applies the same feature engineering as the notebooks, trains a Random Forest, evaluates it on the held-out test set, serialises the model to `models/churn_model.pkl`, and saves evaluation figures to `reports/figures/`.

```bash
python src/train_model.py
```

**What it produces:**
- `models/churn_model.pkl` — trained model
- `reports/figures/confusion_matrix.png`
- `reports/figures/feature_importance.png`
- Console output with accuracy and full classification report

### `src/predict.py`

Loads the saved model and generates predictions for new customer records supplied as a CSV file. Also supports a `--test` flag to re-score the held-out test split.

```bash
# Predict on new data
python src/predict.py --input data/raw/new_customers.csv --output reports/predictions.csv

# Sanity-check against the test split
python src/predict.py --test
```

**Output columns added to input CSV:**

| Column | Description |
|---|---|
| `churn_prediction` | 0 (retained) or 1 (churned) |
| `churn_probability` | Probability of churn [0.0 – 1.0] |

## Key Findings

1. **Class imbalance** — approximately 14.5% of customers churned (483 out of 3,333). Stratified splitting was used to preserve this ratio in training and test sets.

2. **Strongest churn indicators**
   - **Total daytime call minutes** — churned customers exhibit significantly higher daytime usage.
   - **Customer service calls** — customers with 3 or more service calls show dramatically higher churn rates, signalling unresolved complaints.

3. **Service plans reduce churn** — customers with an international plan or a voicemail plan are more likely to be retained, though neither plan is a guarantee.

4. **Account length is weakly predictive** — tenure alone does not reliably distinguish churners from retained customers.

5. **Charge columns are redundant** — all four charge columns are perfectly correlated (r ≈ 1.0) with their minute counterparts and were dropped.

6. **Non-linear models outperform linear ones** — low linear correlation between features and churn confirms that tree-based methods are better suited to this problem. Random Forest achieved the best balance of precision and recall.

## Getting Started

### Prerequisites

- Python 3.9+
- pip

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/<your-username>/customer-churn-prediction.git
cd customer-churn-prediction

# 2. Create and activate a virtual environment (recommended)
python -m venv venv
source venv/bin/activate        # macOS / Linux
venv\Scripts\activate           # Windows

# 3. Install dependencies
pip install -r requirements.txt

# 4. Place the raw dataset
#    Copy churn_dataset.csv into data/raw/
```

### Run the full pipeline

```bash
# Step 1 — open notebooks and run in order
jupyter notebook

# OR — run scripts directly (skips notebooks)

# Step 2 — train the model
python src/train_model.py

# Step 3 — predict on new data
python src/predict.py --input data/raw/new_customers.csv --output reports/predictions.csv
```
